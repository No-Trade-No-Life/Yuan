## 非阻塞建议（Non-blocking）

- **N1（优先级高）**：降低参数“硬编码”风险，避免运维调参改代码

  - 质疑点：`allowed_clock_skew_ms`、幂等窗口 TTL、兼容窗口等关键阈值现在是“默认值 + 文档约束”。
  - 最小化建议：把这些阈值统一声明为配置项（含默认值与上下限），并在 Metrics 暴露当前生效值。

- **N2（优先级中）**：补一条“共享存储不可用”的降级策略

  - 质疑点：nonce/reuse 强依赖共享原子存储；若存储抖动，系统会在安全与可用性之间摇摆。
  - 最小化建议：在 RFC 增加 3 行策略：存储故障时默认 fail-closed、告警升级、允许按环境开只读观测开关。

- **N3（优先级中）**：减少主文长度，保留最小决策正文

  - 质疑点：主 RFC 已承载大量“操作手册细节”，后续演进成本高。
  - 最小化建议：将 canonical 示例扩展、错误码全集、runbook 细步骤外移到附录/独立文档，主文只保留约束与决策。

- **N4（优先级低）**：给 B4 增加“依赖前提”显式清单
  - 质疑点：旧 Host 短窗兼容验签依赖旧链路具备 kid/iss 校验挂载点。
  - 最小化建议：在回滚节补“前置能力检查清单”（3-5 条）即可，不必新增架构。
