# RFC Review Report

## 结论

PASS（可进入门禁）

本轮对抗审查后，RFC 在三项硬门槛上已闭环：

- 可实现：职责单一（调度器写 `state/lease_holder`，执行器仅续租与心跳），切换事务与错误语义可落地。
- 可验证：R55/R59/R60 均有机器可执行判定与测试映射。
- 可回滚：回滚阻断条件与拒绝错误码完整，且开关切换有冻结 cohort 门禁。

- [x] 无（本轮未发现“必须修正”问题）

## Non-blocking

- **[可选优化] `clock_skew_seconds` 保留必要性仍偏弱**
  - 质疑假设：既然过期真值已收敛为 `lease_expire_at`，该参数继续暴露是否会被误读为“判定宽限”？
  - 质疑复杂度：新增一项可配置即新增一类误配置与排障分支。
  - 最小修改建议：文档明确“仅用于观测阈值，不参与 lease 判定”；若一个发布周期内无调优用例，可后续固化常量。
- **[可选优化] selector 语法对人工操作容错偏低**
  - 质疑边界：当前“包含空格即非法”能保证简单性，但会放大手工配置的无效失败。
  - 可替代方案：不扩展语义，仅在解析前 `trim` 分段空白。
  - 最小修改建议：保持 RFC 语义不变，实现层加无副作用预处理并继续对非法表达式报错。
- **[可选优化] R55 回滚失败定位可再降复杂度**
  - 质疑可验证性成本：`NOT EXISTS + 多分支条件` 可执行，但值班场景下定位失败子因子不够直观。
  - 可替代方案：不改判定 SQL，仅补充失败分类映射。
  - 最小修改建议：在实现文档增加“失败条件 -> 错误码 -> 指标标签”表，避免引入新状态机。

## 修复指导

1. 当前无需 must-fix，结论维持 PASS，可进入门禁。
2. 若继续做减法，优先收敛 `clock_skew_seconds` 的文档语义，避免误用。
3. 在实现前补充 R55 失败分类映射表，提升回滚演练与线上排障效率。
